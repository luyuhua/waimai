💡 安装提示:
  • 这是书签工具,不是网页链接,必须拖拽到书签栏(直接点击只会在当前页执行一次)
  • 安装后,在美团商家版任意页面点击书签即可加载面板
  • 面板和日志状态会保存在当前页面,刷新后失效,重新点书签即可
  • 部分浏览器扩展(广告拦截等)可能影响脚本加载,必要时临时关闭

⏰ 出餐计时策略

面板里可以切换 3 种策略,各策略独立计时:

策略 适用场景 默认出餐窗口
下单后
(after_order)		 即时单为主,按备餐节奏出餐 下单后 180 ~ 240 秒
出餐前
(before_deadline)		 建议出餐时长已知的订单 建议出餐前 120 ~ 180 秒
手动出餐
(manual)		 需要人工判断的场景 不自动出餐
💡 预订单处理:预订单会按"建议出餐时间"参与计算,策略与即时单保持一致;只是基准时间从"下单时间"换成"建议出餐时间点"。
⚙️ 调整窗口:面板中策略旁边的数字框可改秒数(范围 0~1800),修改后立即生效,新订单按新窗口计时。

❓ 常见问题

  • 点了书签没反应?大概率是 CSP 拦截,按 F12 看 Console 报错;或检查书签 href 是否被浏览器转义成普通链接。
  • 面板里的订单序号看着乱?V1 已按"状态分组 + 送达时间"双层排序:待出餐/待接单置顶,组内按送达时间升序(最近的在上);其他状态在第二组,组内按送达时间降序。
  • 为什么订单没自动出餐?看日志区:如果只有"操作"日志没"心跳",说明 __monitorCheckCount 计数异常,可刷新页面重试。
  • 想要更完整的功能(分页订单、API 数据)?请使用 V2 版本。
