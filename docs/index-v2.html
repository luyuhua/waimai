⏰ 出餐计时策略

策略即时单预订单
afterOrder
下单后模式		 下单时间 + cookAfterXs ~ cookBeforeYs 虚拟下单时间 + cookAfterXs ~ cookBeforeYs
虚拟下单 = 送达时间 − 20分钟
beforeCook
出餐前模式		 建议出餐时间 − cookAfterXs ~ cookBeforeYs 建议出餐时间 − cookAfterXs ~ cookBeforeYs
（与即时单逻辑相同）
💡 预订单虚拟下单时间：V2 将预订单的虚拟下单时间设为建议出餐前20分钟（V1 为10分钟），给备餐窗口留出更充足时间。

🎮 控制台命令

启动后可在浏览器控制台（F12）使用以下命令：

__WM_START(config?) 启动 V2 监控，可传自定义配置 __WM_STOP() 停止监控 __WM_STATUS() 查看当前运行状态

自定义配置示例：

// 修改出餐策略为"出餐前模式"，出餐前60~600秒自动出餐
__WM_START({
  cook: {
    strategy: 'beforeCook',
    cookAfterXs: 60,
    cookBeforeYs: 600
  }
})
