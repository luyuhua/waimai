### **外卖运营助手 — 完整需求文档**

> 本文档用于指导大模型开发外卖运营助手的 Web UI，涵盖所有功能模块、数据结构、交互逻辑与展示要求。

---

## **一、产品概述**

外卖运营助手是一个面向外卖商家（多平台、多店铺）的运营管理工具，核心能力包括：菜品与成本的基础数据管理、多维度经营数据可视化、AI 驱动的运营报告自动生成。所有统计数据均支持**多平台多店铺的聚合汇总与单独查看**。

---

## **二、全局规则**

### **2.1 多平台多店铺机制**

- 系统支持接入多个外卖平台（美团、淘宝闪购等）和多个店铺
- 所有统计页面顶部提供**平台 + 店铺筛选器**，支持以下选择模式：
  - 全部平台全部店铺（聚合汇总）
  - 指定平台全部店铺
  - 指定平台指定店铺
  - 多选店铺自由组合
- 切换筛选后，页面所有图表和数据同步刷新
- 菜品管理、成本管理等基础数据也按店铺隔离，各店铺独立维护

### **2.2 权限与数据安全**

- 角色：管理员、运营、老板（只读）
- 成本价、净利等敏感数据按角色控制可见性
- 老板角色可查看所有数据但不可编辑

### **2.3 时间筛选**

- 所有数据页面默认提供时间范围选择：今日 / 近7日 / 近30日 / 自定义区间
- 支持同比（去年同期）和环比（上一周期）对比展示

---

## **三、模块一：基础数据管理**

### **3.1 菜品管理**

#### **3.1.1 菜品录入**

每个菜品包含以下字段：

| 字段 | 说明 |
|---|---|
| 菜品名称 | 自有标准名称 |
| 分类 | 如主食、小吃、饮品，支持自定义分类 |
| 成本价 | 单份制作成本（元） |
| 售价 | 平台展示售价（元） |
| 制作流程 | 文本描述，可选填 |
| 备注 | 自由文本，可选填 |
| 所属店铺 | 关联到具体店铺 |
| 状态 | 上架 / 下架 |

- 支持手动逐条创建，也支持批量创建
- 成本价变更时自动记录变更历史（旧值、新值、变更时间），便于追溯毛利变化

#### **3.1.2 平台菜品导入**

- 支持从美团、淘宝闪购等平台导出的数据文件（CSV / Excel）批量导入
- 导入字段至少包含：平台菜品名、平台售价、平台销量
- 导入后进入"待补充"状态，用户可逐条补充成本价、制作流程等自有信息

#### **3.1.3 菜品关联**

- 每个自有菜品可设置**多个别名**（如："红烧肉"的别名可以是"秘制红烧肉"、"经典红烧肉"）
- 平台导入的菜品名通过**别名模糊匹配**自动关联到自有菜品
- 匹配结果分三类展示：
  - ✅ 已自动匹配 → 用户确认即可
  - ⚠️ 多个候选匹配 → 用户选择正确的
  - ❌ 未匹配 → 标记为"待关联"，用户手动指定
- 关联关系可查看、修改、删除；关联后平台菜品数据（销量、售价）自动归入对应自有菜品的统计

#### **3.1.4 菜品维护**

- 支持批量上下架、批量调价（选中多个菜品，统一设置新售价或新成本价）
- 支持按分类筛选、按状态筛选、按店铺筛选
- 菜品列表支持搜索

---

### **3.2 成本管理**

#### **3.2.1 每日运营成本录入**

用户每日手动填写以下运营成本项（按店铺分别录入）：

| 成本项 | 说明 | 录入方式 |
|---|---|---|
| 推广成本 | 平台推广费、竞价排名等 | 每日手动填写金额 |
| 霸王餐成本 | 平台霸王餐活动补贴支出 | 每日手动填写金额 |
| 大额订单补贴 | 大额订单优惠补贴 | 每日手动填写金额 |
| 其他临时成本 | 自定义条目，填写名称+金额+备注 | 每日手动添加 |

- 支持补录历史日期的成本数据
- 每日成本录入页面默认展示当日，可切换日期查看和修改历史记录

#### **3.2.2 固定成本设置**

按店铺设置以下固定成本，设置后每日自动计入，无需重复填写：

| 固定成本项 | 说明 | 设置方式 |
|---|---|---|
| 房租 | 月租金，自动按日摊 | 填写月金额，系统自动日摊 |
| 水电费 | 月水电费，自动按日摊 | 填写月金额，系统自动日摊 |
| 人工成本 | 周期性人工支出 | 填写月金额，系统自动日摊 |
| 其他固定成本 | 自定义条目 | 填写名称+月金额 |

- 固定成本支持修改和暂停（暂停期间不计入）
- 支持查看固定成本列表和摊算明细

## **四、模块二：数据展示**

所有数据来源于：平台订单数据（自动同步） + 自有菜品成本数据 + 用户录入的运营/固定成本数据。

页面顶部统一放置**平台+店铺筛选器**和**时间范围选择器**，切换后全页刷新。

---

### **4.1 营收数据**

按以下层级展示利润结构，每层均可点击展开查看明细：

| 层级 | 计算公式 |
|---|---|
| 营业收入 | 各平台订单总收入之和 |
| 菜品成本 | 各菜品成本价 × 对应销量之和 |
| **毛利** | 营业收入 − 菜品成本 |
| 运营成本 | 推广 + 霸王餐 + 大额补贴 + 其他临时成本 |
| 固定成本 | 房租（日摊） + 水电（日摊） + 人工（日摊） + 其他固定（日摊） |
| **净利** | 毛利 − 运营成本 − 固定成本 |

展示要求：
- 毛利和净利为核心指标，大字突出显示，并展示毛利率和净利率
- 各层级金额以**瀑布图**形式呈现，直观展示从收入到净利的逐层扣减过程
- 支持同比环比对比（如：本周净利 vs 上周净利，变化百分比标红或标绿）
- 运营成本和固定成本分别展示各子项占比饼图
- 关键指标异常时自动标红提醒（如：净利率低于阈值、某项运营成本占比过高）

---

### **4.2 用户数据**

| 数据项 | 展示方式 |
|---|---|
| 新客 / 老客占比 | 饼图 + 趋势折线图 |
| 客单价分布 | 柱状图（按金额区间分档） |
| 订单距离分布 | 柱状图（按距离区间分档，如 0-1km、1-3km、3-5km） |
| 高频客户 | TOP 客户列表（按订单数排序），支持打标签 |

---

### **4.3 订单数据**

| 数据项 | 展示方式 |
|---|---|
| 订单总量 | 数字卡片 + 日趋势折线 |
| 日均订单量 | 数字卡片 |
| 订单时间段分布 | 柱状图或热力图（按小时分段，如 0-6时、6-9时等） |
| 订单取消率 | 数字卡片，异常时标红 |
| 退款率 | 数字卡片，异常时标红 |

---

### **4.4 商品数据**

| 数据项 | 展示方式 |
|---|---|
| 菜品销量排行 | TOP10 柱状图 |
| 菜品毛利排行 | TOP10 柱状图 |
| 销量-毛利交叉分析 | 散点图，四个象限标记：高销量高毛利 / 高销量低毛利 / 低销量高毛利 / 低销量低毛利 |
| 菜品贡献度占比 | 饼图（各菜品毛利占总毛利比例） |

---

### **4.5 通用能力**

- 所有图表支持时间筛选与同比环比对比
- 支持数据导出（CSV / Excel）
- 关键指标异常自动标红提醒

## **五、模块三：运营报告**

### **5.1 自动生成报告**

- **日报**：每日自动生成，内容包含：
  - 当日核心指标卡片（收入、毛利、净利、订单量、客单价）
  - 环比变化与趋势判断
  - 成本明细（运营成本各子项 + 固定成本日摊）
  - TOP 菜品变动（销量上升/下降 TOP3）
  - 异常提醒（指标骤变、成本占比异常等）

- **周报**：每周自动生成，内容包含：
  - 本周核心指标汇总与趋势
  - 周内每日数据对比折线图
  - 用户结构变化分析
  - 成本结构分析
  - TOP 菜品周度排行

### **5.2 AI 运营分析**

- 基于历史数据，AI 自动输出：
  - **趋势判断**：如"近7日净利持续下降，主要受推广成本上升影响"
  - **问题诊断**：如"菜品X销量高但毛利低，建议调整定价或优化成本"
  - **优化建议**：如建议调价菜品、建议淘汰低效菜品、促销时机建议、推广预算优化建议
- 分析结果可一键插入日报/周报
- AI 分析基于净利维度，而非仅毛利

### **5.3 报告管理**

- 报告支持在线查看、导出 PDF
- 支持定时推送（邮件 / 微信 / 钉钉），推送时间可配置
- 支持自定义报告模板（勾选要包含的指标模块，如：是否包含成本明细、是否包含 AI 分析）

## **六、页面结构总览**

整个 Web 应用建议采用以下页面结构：

| 页面 | 所属模块 | 核心内容 |
|---|---|---|
| 菜品列表页 | 基础数据管理 | 菜品 CRUD、批量操作、分类筛选 |
| 菜品导入页 | 基础数据管理 | 平台数据上传、关联匹配、补充信息 |
| 菜品关联页 | 基础数据管理 | 别名管理、自动匹配确认、待关联处理 |
| 每日成本录入页 | 基础数据管理 | 运营成本填写、历史查看与补录 |
| 固定成本设置页 | 基础数据管理 | 固定成本项配置、日摊明细 |
| 营收概览页 | 数据展示 | 利润瀑布图、核心指标卡片、成本占比图 |
| 用户分析页 | 数据展示 | 新老客、客单价、距离分布 |
| 订单分析页 | 数据展示 | 订单量、时段分布、取消退款率 |
| 商品分析页 | 数据展示 | 销量排行、毛利排行、交叉分析 |
| 报告列表页 | 运营报告 | 日报/周报列表、在线查看、导出 |
| 报告详情页 | 运营报告 | 单篇报告完整内容、AI 分析插入 |
| 报告设置页 | 运营报告 | 推送配置、模板自定义 |

## **七、关键交互说明**

1. **全局筛选器**：所有数据页面顶部固定展示「平台选择 + 店铺选择 + 时间范围」，切换后全页数据联动刷新
2. **菜品关联流程**：导入平台数据 → 系统自动匹配 → 用户确认/手动指定 → 关联完成 → 平台数据纳入自有菜品统计
3. **成本录入流程**：每日进入成本录入页 → 填写各项运营成本 → 保存 → 系统自动参与当日净利计算
4. **净利计算链路**：营业收入 → 减菜品成本 → 得毛利 → 减运营成本 → 减固定成本 → 得净利，全链路在营收概览页以瀑布图可视化
5. **报告生成链路**：每日定时触发 → 拉取当日所有数据 → 汇入成本明细 → 生成指标卡片与图表 → AI 分析插入 → 推送或存档
